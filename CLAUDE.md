# CLAUDE.md — сайт Александра Исаева

Контекст проекта для Claude Code. Читать перед любой задачей.

---

## Что это

Двухстраничный статический сайт психолога Александра Исаева + страница-заглушка
`/karta`. Цель: продажа книги «Ты не поломан» и услуг автора (консультации, ретриты).

Метод автора называется **«Древо жизни»** — это имя должно звучать в шапке и в герое
главной. Читатель приходит по адресу, напечатанному в книге, и должен узнать место.

## Стек — жёстко

- Чистый **HTML + CSS + vanilla JS**
- **Никакого** сборщика, npm, фреймворков, React, Astro, Tailwind
- **Никакого** бэкенда, форм, баз данных, приёма платежей, сбора почты
- Один CSS-файл, один JS-файл на весь сайт
- Деплой — `rsync` статики на VPS за Caddy

Если задача выглядит так, будто нужен бэкенд или сборка — остановись и спроси.

## Источники

- `source/*.dc.html` — исходник макета (разметка + SVG + логика)
- `standalone/*.html` — визуальный эталон, **не редактировать**,
  открывать для сверки
- `HANDOFF.md` — дизайн-токены и структура секций
- `CONTENT.md` — все реальные тексты, цены, ссылки. **Ничего не выдумывать**, брать отсюда

## Правила переноса макета

1. Шаблон между `<x-dc>…</x-dc>` → обычный HTML. Плейсхолдеры `{{ … }}` подставить
   значениями из `renderVals()`.
2. Инлайн-стили вынести в `main.css`. В HTML инлайн-стилей не оставлять.
3. SVG (`logo()`, `circles()`, `heroMark()`, `tree()`, `sprig()`, `breathMark()`):
   геометрию, координаты, viewBox и цвета копировать **дословно** из
   `React.createElement`. Ничего не перерисовывать на глаз.
4. Интерактив переписать с React на vanilla JS: древо, аккордеоны FAQ и оглавления,
   hover-эффекты, IntersectionObserver для прорастания древа.
5. `<picture>` / `srcset` / `@media` сохранить как в макете.

## Голос бренда

Спокойный, уверенный, **на «Вы»**, простыми словами, без менторства и гипербол.
Слоган один: «Ты не поломан — ты просто забыл свою силу.»

Отрывок из книги — исключение: он написан на «ты» и оформляется как цитата из книги,
тогда смена обращения читается как приём.

**Запрещено:** кричащие плашки, счётчики, полосы прогресса, «осталось N мест», таймеры,
стоковые улыбки, эмодзи-иконки, красные CTA, градиенты вне палитры, любые сроки выхода
печатного тиража.

## Мобильный приоритет

Основной трафик — офлайн: QR-коды в студии, на ретритах, в PDF-фрагменте. Люди
открывают сайт с телефона стоя в холле. Mobile-first не галочка, а основной сценарий.
Ноль внешних запросов при загрузке.

## Безопасность контента

**Полная книга (`Ты_не_поломан_фирстиль_финал.pdf`, 391 стр.) — это товар.**

- Хранить **вне** папки проекта
- В `assets/files/` кладётся **только** фрагмент на 18 страниц
- Деплой идёт через `rsync --delete` — случайно попавший в `site/` файл уедет
  на публичный сервер и продукт станет бесплатным

## Что не делать без спроса

- Не расширять скоуп: квиз «Карта героя», приём оплаты, сбор почты, бот — не в этом релизе
- Не менять содержание секции «Метод» на главной — древо остаётся как в макете
- Не добавлять счётчики, прогресс-бары и сроки
- Не выдумывать тексты, названия глав, отзывы и цифры — всё в `CONTENT.md`
- Не трогать `standalone/` — это эталон для сверки

## Ритм работы

Один майлстоун из `PLAN.md` за раз. В конце каждого — **стоп-гейт**: отчитаться,
дождаться проверки. Не продолжать автономно на следующий майлстоун.
